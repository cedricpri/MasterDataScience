Scalable kernel methods¶

The main objective of this exercise consists in creating a script able to download hml data in order to predict houses price using kernel ridge regression.

For this, several steps will be followed: first of all, the data will be downloaded and shortly analyzed. Then, once divided into a test and a train dataset, a linear regression will be performed in order to predict the price of the house based on 16 different input characteristics. Final, a kernel ridge regression method will be trained, using the Nyström or RFF approximation and the mean absolute error obtained by both methods will be compared.

Dataset division¶

The next step consists in dividing our dataset into train (2/3) and test (1/3). This is quite easy to do thanks to the sklearn package. Before doing so, the first step is to normalize the house price in order to avoid avoir huge numbers for the error and any potential issue with the regression (this is especially true if we use the MSE instead of the MAE, as done in this exercise).

In [180]:
from sklearn.preprocessing import StandardScaler #To normalize the data
from sklearn.model_selection import train_test_split

normalizedData = StandardScaler().fit_transform(data.data)
normalizedTarget = StandardScaler().fit_transform(data.target.reshape(-1, 1))

#Let's use the random_state option for this exercise to always get the same splitting when dividing the data
X_train, X_test, y_train, y_test = train_test_split(normalizedData, normalizedTarget, test_size=0.33, random_state=0)

Linear regression¶

We are now ready for the analysis itself. Let's start by performing a linear regression on this data, to study the target variable (house price) with respect to the 16 features previously described. We can also compute at this stage the Mean Absolute Error (MAE) on the test sample.

from sklearn.metrics import mean_absolute_error
from sklearn.preprocessing import StandardScaler

y_pred_lin_reg = lin_reg.predict(X_test)

#Calculate the mean absolute error

Kernel Ridge Regression¶

Now, let's use the KRR method in order to see if we can improve this result, using the Nystrom and RFF approximations. A gridsearch is performed in order to find the best gamma and n_components parameters in both cases. This grid search will not be performed on the n_components parameter in order to speed up the code and since we know anyway the having a higher value is always better for the prediction, even though it results in an increased computing time in the same time. A compromise needs to be found.

In [182]:

When repeating the same exercise using non-normalized datasets, we obtain an MAE of 28282 (gamma = 0.001) for the Nystrom approximation and 29339 (gamma = 0.001) for the RFF approximation (however, the gamma parameters set in this case for the grid search are probably not optimal).
